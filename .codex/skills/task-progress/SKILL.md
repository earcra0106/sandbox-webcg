# Task Progress

作業を後から再開できるように、現在の状態を `temp/task.md` に記録する。

## 手順

1. `git status --short --branch` で現在のブランチと変更状態を確認する。
2. 必要に応じて、作業内容に関係するファイルの差分や内容を確認する。
3. `temp/` がなければ作成する。
4. `temp/task.md` を作成または更新する。

## 記録内容

`temp/task.md` には、次の内容を簡潔に記録する。

- 作業タイトルまたは依頼内容。
- 使った skill。
- 実施済みの作業。
- 変更済みまたは追加済みのファイル。
- 現在の `git status --short --branch` の要約。
- 意図的にコミット対象から外すファイルがある場合は、その理由。
- 未実施の作業。
- 次に再開するときの具体的な手順。

## 注意

- `temp/task.md` は再開用の一時メモとして扱う。
- `temp/` が `.gitignore` されている場合、`temp/task.md` を `git add` しない。
- ユーザーが明示的に求めない限り、進行状況の記録だけを理由に commit しない。
