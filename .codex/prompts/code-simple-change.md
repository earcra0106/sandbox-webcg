# code-simple-change

以下の手順で、単純なコード変更を行ってください。

## 目的

ブランチ作成や `git add` を行わず、ユーザーに依頼された小さなコード変更だけを実施します。

## 適用条件

このコマンドは、以下をすべて満たす場合に使ってください。

- 変更範囲が明確で小さい。
- 既存ファイルへの軽微な修正である。
- ブランチ作成、タスクファイル作成、`git add`、コミットが不要であるとユーザーが明示または文脈上期待している。

## 禁止事項

```bash
git checkout -b <branch-name>
git add <changed-files>
git commit -m "<message>"
git push
```

## 実装ルール

- 既存の構成、命名、実装パターンを優先してください。
- 依頼範囲外のリファクタリングは行わないでください。
- WebGPU を触る場合は `navigator.gpu` の存在確認と未対応時の表示を維持してください。
- WebGL を触る場合は WebGL 2 優先の方針を維持してください。

## 完了確認

必要最小限の検証を行ってください。実行できるテストや型チェックがある場合は、変更の規模に応じて実行してください。

```bash
git diff
git status --short --branch
```
