# code-implementation

以下の手順で、コード実装作業を行ってください。

## 目的

ブランチ作成、タスク記録、コード実装、検証、`git add`、コミットまでを含む通常の実装作業を行います。

## 前提

作業前に `/requirements` を実行し、作業ブランチと `docs/tasks/` の記録を作成してください。

## 実装ルール

- 既存の構成、命名、実装パターンを優先してください。
- WebGPU は `navigator.gpu` の存在確認と未対応時の表示を用意してください。
- WebGL は特別な理由がなければ WebGL 2 を優先してください。
- シェーダーコードは、GLSL と WGSL の対応関係が分かるように配置またはコメントしてください。
- サンプルを追加した場合は、目的、操作方法、対応 API、既知の制限をドキュメントに追記してください。
- UI は描画領域とパラメータ操作を優先し、実験結果を確認しやすい構成にしてください。

## 検証

プロジェクトに設定されている検証コマンドを確認し、必要に応じて実行してください。

```bash
git status --short --branch
```

ビルド、型チェック、lint、テストが設定されている場合は実行し、実行できない場合は理由を記録してください。

## 完了手順

変更内容を確認し、適切な単位でステージしてコミットしてください。

```bash
git diff
git add <changed-files>
git commit -m "<変更内容がわかる具体的な日本語メッセージ>"
```

`git push` などのリモート操作は、ユーザーが明示的に許可または依頼した場合のみ行ってください。
