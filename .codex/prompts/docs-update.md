# docs-update

以下の手順で、ドキュメント更新作業を行ってください。

## 目的

ブランチ作成、タスク記録、ファイル編集、`git add`、コミットまでを含む通常のドキュメント更新を行います。

## 前提

作業前に `/requirements` を実行し、作業ブランチと `docs/tasks/` の記録を作成してください。

## 更新ルール

- 実行環境に関する情報提供があった場合は、`docs/specs/` に記載してください。
- 環境構築中にインストールした主要パッケージは、`docs/specs/` に記載してください。
- トークンの肥大化を避けるため、各パッケージごとに記載ファイルを分割してください。
- `docs/PROJECT_SPEC.md` は `docs/specs/` 配下の目次として更新してください。
- 各コマンドでしか使わないルールは `AGENTS.md` に追記せず、該当する `.codex/prompts/*.md` に記載してください。

## 完了手順

変更内容を確認し、適切な単位でステージしてコミットしてください。

```bash
git status --short --branch
git diff
git add <changed-files>
git commit -m "<変更内容がわかる具体的な日本語メッセージ>"
```

`git push` などのリモート操作は、ユーザーが明示的に許可または依頼した場合のみ行ってください。
