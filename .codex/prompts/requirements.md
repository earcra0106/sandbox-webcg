# requirements

以下の手順で、作業前の要件定義を行ってください。

## 目的

これから行う作業の目的、スコープ、実装方針を明確にし、`docs/tasks/` に作業記録を作成します。

## 手順

1. 作業内容を確認し、作業タイトルを決めてください。
2. 作業タイトルは git branch 名としても使えるよう、以下のルールで命名してください。
    - 作業分類ごとに `fix/` `feature/` `refactor/` `env/` のいずれかの接頭辞を付ける。
    - アンダースコア `_` は使わない。
    - 単語の区切りにはハイフン `-` を使う。
3. 現在のブランチが `main` であることを確認してください。
4. 作業タイトルをもとに、作業ブランチを作成してください。
5. `docs/tasks/` に要件定義を記録してください。

## 実行コマンド

```bash
date +"%Y%m%d_%H%M"
git branch --show-current
git checkout -b <branch-name>
```

## タスクファイル

ファイル名は `yyyyMMdd_hhmm_タイトル.md` としてください。ここではタイトルに作業分類の接頭辞を含めないでください。

記載内容:

- 作業タイトル
- 作業の目的と概要
- 作成・編集するファイルのスコープ
- 実装する関数、クラス、モジュールの概要
- 実装した機能にアクセスするためのURLパスやUI操作の説明

ドキュメント更新のみの作業では、作業方針の記述のみで構いません。

## 例外

ブランチ作成に失敗した場合は、`docs/issues/` に原因と代替案を記録し、ユーザー確認の上で作業を継続してください。
