# 実装ガイドライン

- 初期化、リソース作成、描画ループ、破棄処理を分離し、比較しやすい構成にする。
- WebGL と WebGPU の差分を隠しすぎない。共通化は、実験の意図を損なわない範囲に留める。
- シェーダーは外部ファイル化するか、少なくとも実装ファイル内でまとまった定数として扱う。
- ブラウザ API の失敗は握りつぶさず、画面またはコンソールに原因が分かる形で出す。
- フレームレート、解像度、デバイスピクセル比など、描画結果に影響する値は確認しやすくする。
- 関数を作成する場合は、まず対応する spec ファイルを作成し、通ることを確認してから実装する。
- 新しい主要ライブラリを導入した場合は、導入した技術名と導入目的を `docs/specs/` に記録する。
