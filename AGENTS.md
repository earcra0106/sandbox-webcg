# Codex Notes

このリポジトリは、WebGL と WebGPU を使ってウェブページ上で CG を描画するための実験プロジェクトです。

## プロジェクト方針

- WebGL と WebGPU の違いを比較しやすい実験コードを蓄積する。
- 1つの描画テーマに対して、可能な範囲で WebGL 版と WebGPU 版を並べて実装する。
- 描画ロジック、シェーダー、初期化処理、リソース管理の違いが追いやすい構成にする。
- 実験コードであっても、ブラウザでの動作条件、制限、失敗時の挙動を明記する。

## 実装時の注意

- WebGPU は対応ブラウザや実行環境の制約があるため、`navigator.gpu` の存在確認とフォールバック表示を用意する。
- WebGL は WebGL 2 を優先し、必要があれば WebGL 1 との差分を明記する。
- シェーダーコードは、GLSL と WGSL の対応関係が分かるように配置またはコメントする。
- サンプルを追加した場合は、目的、操作方法、対応 API、既知の制限をドキュメントに追記する。
- UI は実験結果を確認するためのツールとして扱い、過度な装飾よりも描画領域とパラメータ操作を優先する。

## 作業ルール

作業中には以下のルールを守ること。

1. 作業内容に応じて、`.codex/skills/` の skill を参照すること。
    - 要件定義: `$requirements`
    - docs更新: `$docs-update`
    - コード実装: `$code-implementation`
    - docs更新の単純なファイル変更: `$docs-simple-change`
    - コード実装の単純なファイル変更: `$code-simple-change`
    - 作業完了後にmainブランチを最新化する: `$done`
2. 各 skill でしか使わないルールは `AGENTS.md` に追記せず、該当する `.codex/skills/<skill-name>/SKILL.md` に記載すること。
3. `git push` などのリモート操作は、ユーザーが明示的に許可または依頼した場合のみ行うこと。
4. 作業中にバグが発生した場合は、タイトル、発生日時( `yyyy/MM/dd hh:mm` )、分類、原因、解決方法、解決できない場合は代替案などを記載したレポートを作成すること。発生日時は `date +"%Y/%m/%d %H:%M"` コマンドで取得すること。
    - 作成したレポートを `docs/issues/` フォルダの下に、 `yyyyMMdd_hhmm_タイトル.md` として配置すること。
    - バグの分類は以下から選択し、適切な表現がなければ、この文書に追加すること。
        - `env` : 環境構築や実行環境に関する問題
        - `build` : ビルドプロセスや依存関係に関する問題
        - `code` : コードのバグやロジックエラー
        - `performance` : パフォーマンスの問題
        - `compatibility` : ブラウザやプラットフォーム間の互換性の問題
        - `other` : 上記以外の問題

## 既知の環境制約

- この実行環境では、`.git/HEAD.lock` や `.git/index.lock` の作成が読み取り専用ファイルシステムとして拒否されることがある。ブランチ作成、チェックアウト、コミットなど `.git/` のメタデータ更新を伴う操作が失敗した場合は、同じ通常権限のコマンドを繰り返さず、権限昇格またはユーザー側での実行に切り替えること。
- この実行環境には Python 実行環境がない前提で作業する。skill を追加・更新する場合は、Python ベースの補助スクリプトに依存せず、既存 skill の構成を参考に `SKILL.md` と `agents/openai.yaml` を直接編集すること。
